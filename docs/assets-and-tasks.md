# Assets & Tasks

HMM is built around maintenance tasks. A task is the work that needs to be done. An asset is the real-world item the task belongs to.

## Assets

Assets are optional but useful for organizing tasks around equipment, rooms, or systems.

Examples:

- HVAC system
- Pool pump
- Hot tub
- Water softener
- Refrigerator
- Lawn mower
- Snowblower
- Generator

## Tasks

A task describes a recurring maintenance action.

Examples:

- Replace HVAC filter
- Clean salt cell
- Drain and refill hot tub
- Lubricate garage door
- Clean dryer vent
- Replace refrigerator water filter

## Recommended naming

Use an action-oriented task name:

```text
Replace HVAC filter
Clean pool cartridge filter
Test sump pump
Inspect hot tub cover
```

Avoid vague names such as:

```text
HVAC
Pool
Filter
Maintenance
```

## Categories

Categories help group similar work. Examples include:

- HVAC
- Plumbing
- Electrical
- Appliances
- Pool / Hot Tub
- Lawn / Garden
- Vehicle
- Safety
- Seasonal

## Task lifecycle

A task normally moves through this lifecycle:

1. Created
2. Upcoming
3. Due
4. Overdue, if not completed
5. Completed
6. Recalculated for the next due cycle

Seasonal tasks may also become paused while outside their active window.

## Best practices

- Start with a small number of high-value tasks.
- Use assets when several tasks belong to the same equipment.
- Use NFC tags for equipment where the task is usually completed at the equipment location.
- Use runtime and metered schedules only when the source sensor is reliable.
