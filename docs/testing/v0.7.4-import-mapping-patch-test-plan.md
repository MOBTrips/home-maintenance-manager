# v0.7.4 Import Mapping Patch Test Plan

This patch fixes reviewed imports so task-by-task entity configuration is truly task-specific. It must not change the Task Pack schema, import/export compatibility, dashboard, task list, task detail dialog, task editor, history screen, or unrelated backend behavior.

## Scope

- Keep existing global `entity_mapping` support for older import clients.
- Add reviewed import support for task-specific entity mappings keyed by task ID and placeholder.
- Prefer task-specific mappings during backend import apply when available.
- Allow two selected tasks using the same placeholder to map to different real Home Assistant entities.
- Keep required entity blocking per task.
- Keep optional skipped entities scoped to the specific task.

## Manual Checks

1. Prepare a Task Pack where two selected tasks both reference the same placeholder, such as `hmm://entity/runtime_source`.
2. Open the import wizard and select both tasks.
3. In Configure Tasks, map the first task's placeholder to one real HA entity.
4. Map the second task's same placeholder to a different real HA entity.
5. Confirm the wizard does not show a mapping conflict and allows Review Import.
6. Confirm Review Import shows each task with its own assignment.
7. Apply the import and confirm each imported task saved its own mapped entity.
8. Repeat with one optional placeholder skipped on one task and mapped on another task; confirm the skip only affects the skipped task.
9. Repeat with an unresolved required placeholder on one task; confirm import is blocked until that task is mapped.

## Compatibility Checks

1. Run an older-style import apply payload with only global `entity_mapping` and no `task_entity_mapping`; confirm placeholders still map correctly.
2. Confirm Task Pack JSON schema is unchanged.
3. Confirm backup-style imports still accept existing entity mapping behavior.
4. Confirm exported Task Packs still import through the refreshed wizard.
