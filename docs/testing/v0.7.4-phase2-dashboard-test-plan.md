# v0.7.4 Phase 2 Dashboard Test Plan

Phase 2 refreshes the dashboard only. It must preserve task data, filters, actions, imports, exports, dialogs, task editor behavior, history behavior, and import wizard behavior.

## Scope

- Home Health dashboard card using existing task status data.
- Attention Summary section with Overdue, Due Today, Due Soon, and Upcoming counts.
- Improved dashboard spacing and section hierarchy.
- Existing category cards and Next up task feed remain functional.

## Manual dashboard checks

1. Open the HMM dashboard and confirm it renders without console errors.
2. Confirm the dashboard sections appear in this order:
   - Dashboard / Home Health
   - Attention Summary
   - Categories
   - Next up
3. Confirm the Home Health score starts at 100 for an empty or fully on-track task set.
4. Confirm overdue tasks reduce the Home Health score more heavily than due-today tasks.
5. Confirm the score never renders below 0 or above 100.
6. Confirm Attention Summary counts match current task statuses:
   - `overdue` appears under Overdue.
   - `due` appears under Due Today.
   - `upcoming` appears under Due Soon.
   - `ok` appears under Upcoming.
7. Confirm category cards still open the Tasks tab with the selected category filter.
8. Confirm Next up task cards still support Mark complete, Snooze 7 days, View, Edit, and Open HA Device where available.
9. Confirm the dashboard empty state still offers Add maintenance task when no tasks exist.
10. Check desktop and mobile widths for readable spacing, no overlap, and usable buttons.

## Non-dashboard regression checks

1. Open and close task detail dialogs from the refreshed dashboard.
2. Open the task editor from a dashboard task card and confirm fields and save behavior are unchanged.
3. Open Settings and confirm import/export controls still render.
4. Open the import wizard and confirm existing steps still navigate.
5. Open History and confirm it still renders the existing history list.

## Validation checks

Run:

```bash
python3 -m compileall custom_components/home_maintenance_manager
```

Run the frontend parse check with the bundled or local Node runtime:

```bash
node --check custom_components/home_maintenance_manager/frontend/home-maintenance-manager-panel.js
```

Run the existing test suite if `pytest` is available:

```bash
python3 -m pytest
```
