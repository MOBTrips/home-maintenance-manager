# v0.7.4 Phase 7 Import Wizard Test Plan

Phase 7 refreshes the import wizard UI only. It must not change backend behavior, task-pack schema, import/export compatibility, dashboard, task list, task detail dialog, task editor, or history screen.

## Scope

- Import wizard uses four steps: Select Tasks, Configure Tasks, Review Import, and Import Complete.
- Task selection remains the first user decision.
- Entity configuration is task-by-task instead of grouped by shared entity requirement.
- Native Home Assistant entity pickers are used for entity selection.
- Suggested entity cards are not shown in the import wizard.
- Required entities block review/import until selected.
- Optional entities may be skipped.

## Select Tasks Checks

1. Open the import wizard from Settings with a backup-style JSON export.
2. Open the import wizard from a Task Pack JSON file.
3. Open the import wizard from a built-in Task Pack.
4. Confirm Select Tasks shows package context, task counts, entity counts, and task rows.
5. Confirm task status filters still work.
6. Confirm Select all valid and Select none still work.
7. Confirm selected tasks remain selected after navigating forward and backward.

## Configure Tasks Checks

1. Select multiple tasks that have missing entity requirements.
2. Confirm the Configure Tasks step shows progress such as Task 1 of N.
3. Confirm each selected task is configured independently.
4. Confirm task name, category, schedule context, and description/context are visible while selecting entities.
5. Confirm each missing entity requirement shows role, required/optional status, placeholder, domain, device class, state class, and unit when available.
6. Confirm native Home Assistant entity pickers appear for each requirement.
7. Confirm entity picker domain filtering is applied when requirement metadata provides domains.
8. Confirm suggested entity cards are not shown.

## Required And Optional Checks

1. Leave a required entity unselected and confirm Review Import cannot be reached.
2. Select all required entities and confirm Review Import can be reached.
3. Skip an optional entity and confirm Review Import can still be reached.
4. Clear a selected required entity and confirm Review Import becomes blocked again.
5. Confirm optional skipped entities appear as skipped in Review Import.

## Review And Apply Checks

1. Confirm Review Import shows selected tasks and task-by-task entity assignments.
2. Confirm Merge/Replace options still appear for backup-style imports.
3. Confirm Task Packs still force merge mode.
4. Confirm restore-deleted and import-settings controls preserve existing behavior.
5. Confirm Import is disabled when required entities are incomplete.
6. Confirm Import succeeds with valid mappings.
7. Confirm Import Complete shows new tasks, updated tasks, paused tasks, skipped tasks, and total tasks.

## Compatibility Checks

1. Confirm selected-task import behavior is preserved.
2. Confirm required vs optional entity behavior is preserved.
3. Confirm backend import preview and apply payloads remain compatible.
4. Confirm Task Pack schema is unchanged.
5. Confirm exported Task Packs still import through the refreshed wizard.

## Non-Wizard Regression Checks

1. Confirm dashboard still renders.
2. Confirm task list Comfortable and Compact modes still render.
3. Confirm task detail dialog still opens.
4. Confirm task editor still creates and edits tasks.
5. Confirm History screen still renders.
6. Confirm JSON export and Task Pack export still work.

## Layout Checks

1. Verify desktop wizard layout is readable across all four steps.
2. Verify mobile wizard layout stacks cleanly.
3. Verify entity picker cards remain usable on mobile.
4. Verify sticky footer actions remain reachable.
