# v0.7.4 Phase 8 UI Regression Test Plan

Phase 8 polishes the v0.7.4 UI refresh only. It must not add features, change backend behavior, change task-pack schema, add Maintenance Rule #2 backend support, add Seasonal Restriction backend support, or change import/export compatibility.

## Scope

- Review dashboard, task list Comfortable mode, task list Compact mode, task detail dialog, task editor, History timeline, and import wizard together.
- Fix only visual consistency issues such as spacing, alignment, wrapping, status chip consistency, button placement, empty states, overflow, and dialog sizing.
- Remove stale CSS or helper code introduced during v0.7.4 UI phases when it is no longer rendered.
- Preserve existing task data, filters, actions, imports, exports, dialogs, and calculations.

## Dashboard Checks

1. Confirm Dashboard loads without console errors.
2. Confirm Home Health, Attention Summary, dashboard metrics, and category summaries are readable on desktop.
3. Confirm the dashboard stacks cleanly on mobile with no horizontal overflow.
4. Confirm dashboard cards do not change task data, filters, imports, exports, or dialogs.

## Task List Checks

1. Confirm Comfortable mode renders task cards with status chips, progress, dates, and actions.
2. Confirm Complete, View, Edit, Delete, NFC, and Open HA Device actions still work where applicable.
3. Confirm Compact mode renders dense rows with status, category icon, task name, progress, and primary action.
4. Confirm Compact mode shows more tasks on screen than Comfortable mode.
5. Confirm the selected view mode survives page reload.
6. Confirm search, category filter, status filter, and sort behavior still work in both modes.
7. Confirm long task names and status labels do not overflow on mobile.

## Task Detail Checks

1. Open task details from Comfortable cards.
2. Open task details from Compact rows.
3. Confirm Summary, Schedule, Runtime Tracking, Reminders, NFC, Notes, and History sections remain readable.
4. Confirm Complete Task remains highly visible and still completes the task.
5. Confirm Edit and Delete actions preserve existing behavior.
6. Confirm NFC and runtime/entity information appears when available.
7. Confirm the dialog fits desktop and mobile viewports without hidden footer actions.

## Task Editor Checks

1. Create a new task using the refreshed editor.
2. Edit an existing task and save without changing unrelated fields.
3. Confirm schedule settings still save correctly.
4. Confirm runtime and metered entity settings still save correctly.
5. Confirm reminders still save correctly.
6. Confirm NFC settings still save correctly.
7. Confirm Delete behavior is unchanged.
8. Confirm Maintenance Rule #2 and Seasonal Restrictions remain placeholders only.

## History Checks

1. Confirm History loads existing completed task data.
2. Confirm entries are grouped by date.
3. Confirm history filters still work.
4. Confirm the empty history state is clear.
5. Confirm long task names and notes wrap cleanly on mobile.

## Import Wizard Checks

1. Open the import wizard with a backup JSON export.
2. Open the import wizard with a Task Pack JSON file.
3. Open the import wizard from a built-in Task Pack.
4. Confirm selected tasks stay selected while navigating between steps.
5. Confirm entity configuration remains task-by-task.
6. Confirm required unresolved entities prevent review/import.
7. Confirm optional unresolved entities can be skipped.
8. Confirm Review Import clearly shows selected tasks and entity assignments.
9. Confirm a valid Task Pack import completes successfully.
10. Confirm import/export and task-pack schema compatibility are unchanged.

## General UI Checks

1. Navigate all refreshed screens without console errors.
2. Confirm there are no dead buttons in normal navigation.
3. Confirm status chips use consistent colors, icons, sizing, and wrapping.
4. Confirm buttons are aligned and reachable on mobile.
5. Confirm dialogs do not exceed the viewport without scroll support.
6. Confirm empty states are intentional and not visually broken.
7. Confirm no obvious horizontal overflow appears at mobile widths.
