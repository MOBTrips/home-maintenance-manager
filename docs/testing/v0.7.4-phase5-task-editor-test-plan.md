# v0.7.4 Phase 5 Task Editor Test Plan

Phase 5 refreshes the task editor layout only. It must not change backend behavior, saved task data shape, task deletion, dashboard behavior, task lists, task details, history, imports, exports, task packs, or the import wizard.

## Scope

- Task editor sections are reorganized into Basics, Maintenance Rule #1, Maintenance Rule #2, Seasonal Restrictions, Reminders, NFC, Entity Tracking, and Advanced.
- Maintenance Rule #1 maps to the existing schedule fields.
- Maintenance Rule #2 is informational only for this phase.
- Existing seasonal, reminder, NFC, entity tracking, baseline, and instruction fields keep their current behavior.

## Create Task Checks

1. Create a time interval task with name, category, description, baseline, and instructions.
2. Create a runtime task with a runtime source, runtime interval, and running method.
3. Create a metered usage task with a meter source, amount, target unit, and source type.
4. Create a calendar task with both supported calendar patterns.
5. Confirm no new required fields are introduced.
6. Confirm the new Maintenance Rule #2 placeholder does not save new data or block saving.

## Edit Task Checks

1. Open an existing task from a Comfortable task card and save without changes.
2. Open an existing task from a Compact task row and save without changes.
3. Change the task name, category, description, and instructions, then confirm the saved task updates.
4. Change the current schedule fields in Maintenance Rule #1 and confirm the task saves correctly.
5. Change runtime/entity schedule settings and confirm the task saves correctly.
6. Change metered usage settings and confirm the task saves correctly.
7. Change baseline options and confirm due/progress behavior remains consistent with the existing editor.

## Seasonal Restrictions Checks

1. Save a task with seasonal restrictions disabled.
2. Save a task with one selected preset season.
3. Save a task with multiple selected preset seasons.
4. Save a task with a custom active date range.
5. Confirm existing inactive display and usage tracking options still save.
6. Confirm validation still warns when seasonal restrictions are enabled without a season or custom range.

## Reminders And NFC Checks

1. Save a task using global notification behavior.
2. Save a task with task-level notification override and mobile target override.
3. Save a task with notifications disabled.
4. Save a task with an NFC tag and each existing NFC action option.
5. Confirm NFC assignment behavior remains unchanged.

## Delete Checks

1. Open an existing task in the refreshed editor.
2. Click Delete and cancel the confirmation.
3. Click Delete again and confirm.
4. Confirm the existing deletion behavior and tombstone behavior remain unchanged.

## Non-Editor Regression Checks

1. Confirm the Dashboard still renders and existing dashboard actions work.
2. Confirm the Tasks tab Comfortable and Compact modes still render.
3. Confirm the task detail dialog still opens and its actions still work.
4. Confirm the History screen still renders.
5. Confirm Settings import/export controls still render.
6. Confirm the import wizard still opens, navigates, previews, and applies only after confirmation.

## Layout Checks

1. Verify desktop layout is easier to scan and section headings are clear.
2. Verify mobile layout stacks cleanly with no overlapping text, controls, or footer actions.
3. Verify Home Assistant entity pickers remain usable in the refreshed editor.
4. Verify the Save/Create and Delete actions remain visible and unchanged.

## Validation Checks

Run:

```bash
python3 -m compileall custom_components/home_maintenance_manager
```

Run the frontend parse check with the bundled or local Node runtime:

```bash
node --check custom_components/home_maintenance_manager/frontend/home-maintenance-manager-panel.js
```

Run the existing test suite if `pytest` is available:

```bash
python3 -m pytest
```
