# v0.7.4 RC Bugfix Test Plan

This plan covers only the remaining v0.7.4 release candidate bug fixes. It must not validate Maintenance Rule #2 backend support, Task Pack schema changes, import/export schema changes, or unrelated UI redesign work.

## Bug 5A: Meter Source Type On Create

1. Create a new task.
2. Select Metered usage.
3. Select a meter source entity.
4. Select "Reset/session counter - add positive deltas."
5. Save the task.
6. Reopen the task editor.
7. Confirm "Reset/session counter - add positive deltas" remains selected.
8. Repeat with "Cumulative meter - already total" and confirm cumulative behavior is unchanged.

## Bug 5B: Metered Usage Units

1. Create or edit a metered usage task whose source sensor reports seconds.
2. Configure the editor interval in minutes, hours, or days.
3. Save and reopen the task.
4. Confirm the editor display unit remains selected.
5. Confirm the generated usage used, usage remaining, and totalized usage sensors use the configured display unit.
6. Confirm values convert correctly:
   - 60 minutes equals 3600 seconds.
   - 2 hours equals 7200 seconds.
   - 1 day equals 86400 seconds.

## Bug 5C: Session Counter Accumulation

1. Configure a metered usage task as a reset/session counter.
2. Use a source sensor sequence like `0 -> 15 -> 30 -> 75 -> 150 -> 0`.
3. Confirm positive increases are added to the internal totalizer.
4. Confirm the reset to zero does not subtract usage.
5. Start a second cycle such as `0 -> 10 -> 40`.
6. Confirm the second cycle adds to the existing total.
7. Confirm Metered Usage Used increases, Remaining decreases, and Totalized Usage is valid.

## Bug 8: Import Configure Tasks Scroll

1. Open a Task Pack or backup import with multiple selected tasks requiring configuration.
2. Go to Configure Tasks.
3. Scroll into the task configuration area.
4. Move to the next task from the task list or Previous/Next task buttons.
5. Confirm the wizard keeps the user in the task configuration area instead of jumping to the top.
6. Repeat on desktop and mobile viewport widths.

## Regression Checks

1. Confirm task-specific entity mapping still allows two selected tasks sharing a placeholder to map to different HA entities.
2. Confirm existing global `entity_mapping` imports still work.
3. Confirm import/export compatibility is unchanged.
4. Confirm Task Pack schema is unchanged.
