# Single Codex Prompt: HMM v0.7.4 UI Refresh

Implement the HMM v0.7.4 UI Refresh using the uploaded latest source.

Follow these project rules:

1. Work incrementally and preserve existing behavior.
2. Keep storage and task pack schema compatibility unless absolutely required.
3. Do not implement deferred features such as AI recommendations, weather scheduling, table view, or analytics engine.
4. Do not break import/export, task packs, deletion, NFC, or existing task rendering.
5. Update docs and changelog.
6. Use Home Assistant native controls and CSS variables where practical.

Add the design source-of-truth document:

- `docs/design/ui-design-spec-v3.1.md`

Then implement the UI refresh in this order:

1. UI foundation styles and reusable visual patterns.
2. Dashboard refresh with Home Health plus Things Requiring Attention.
3. Comfortable/Compact task views with remembered preference.
4. Task card/row refresh with stronger status icons/colors/progress bars.
5. Task editor refresh using Maintenance Rule #1 and Maintenance Rule #2 language.
6. Seasonal Restrictions directly under Maintenance Rules.
7. History timeline layout.
8. Import wizard redesign:
   - Steps: Select Tasks, Configure Tasks, Review Import, Import.
   - Task-by-task entity configuration.
   - No grouped/shared entity mapping UI.
   - No suggested entity list.
   - Use native HA entity picker filtered by relevant domains.
9. Mobile polish and regression testing.

Key acceptance criteria:

- Dashboard answers: what needs attention and how healthy is my home.
- Compact mode shows many tasks in dense rows and preserves progress indicators.
- Task editor no longer presents combo schedule types as the primary UX; it uses Rule #1 and optional Rule #2.
- Completing a task remains a single completion event and resets all active rules.
- Seasonal Restrictions are visibly separate from schedule rules and placed immediately below them.
- Import entity selection is performed in task context, one task at a time.
- Existing import/export, NFC, and task pack tests continue to pass.

Run the existing test suite and report any failures with likely cause.
