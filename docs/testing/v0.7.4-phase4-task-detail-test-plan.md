# v0.7.4 Phase 4 Task Detail Test Plan

Phase 4 refreshes the task detail/view dialog only. It must not change backend behavior, task editor layout, edit/delete behavior, history screen behavior, imports, exports, task packs, or the import wizard.

## Scope

- Task detail dialog becomes an informational view.
- Existing detail actions remain available.
- Detail content is organized into Summary, Schedule, Runtime Tracking, Reminders, NFC, Notes, and History.
- Complete Task remains highly visible.

## Opening checks

1. Open a task detail dialog from a Comfortable task card.
2. Open a task detail dialog from a Compact task row.
3. Open a task detail dialog from an NFC assigned task row if available.
4. Confirm the dialog closes with the header Close button, footer Close button, and scrim click.

## Action checks

1. Click Complete Task and confirm the existing complete behavior still runs.
2. Click Snooze 7 days and confirm the existing snooze behavior still runs.
3. Click Open HA Device on a task with a generated device and confirm the existing navigation behavior still runs.
4. Click Edit task and confirm the existing task editor opens.
5. Confirm Delete behavior is unchanged and remains available only in the existing task editor.

## Content checks

1. Confirm Summary shows status, category, asset, progress, next due, last completed, completion count, and area when available.
2. Confirm Schedule shows existing schedule progress data.
3. Confirm Runtime Tracking shows runtime/meter entities, current state, detection, baseline, or linked entities when available.
4. Confirm Reminders shows task notification behavior and mobile target override.
5. Confirm NFC shows action, tag IDs, and last scan when available.
6. Confirm Notes shows description and instructions when available.
7. Confirm History shows completion history and recent activity when existing data is available.
8. Confirm empty informational states render clearly when optional data is not configured.

## Layout checks

1. Verify desktop layout is readable and sections do not look like editable form fields.
2. Verify mobile layout stacks cleanly with no overlapping text or buttons.
3. Verify Complete Task remains visible in the summary and footer on desktop and mobile.

## Non-detail regression checks

1. Confirm the Tasks tab Comfortable/Compact toggle still works.
2. Confirm the Dashboard still renders.
3. Confirm the History screen still renders.
4. Confirm Settings import/export controls still render.
5. Confirm the import wizard still opens and navigates.

## Validation checks

Run:

```bash
python3 -m compileall custom_components/home_maintenance_manager
```

Run the frontend parse check with the bundled or local Node runtime:

```bash
node --check custom_components/home_maintenance_manager/frontend/home-maintenance-manager-panel.js
```

Run the existing test suite if `pytest` is available:

```bash
python3 -m pytest
```
