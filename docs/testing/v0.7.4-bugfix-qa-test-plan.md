# v0.7.4 Bugfix QA Test Plan

This plan covers targeted release QA bug fixes only. It must not validate new features, schema changes, or unrelated UI redesign work.

## Scope

- Task editor Maintenance Rule #2 placeholder copy.
- Compact task view progress bar fill.
- Most urgent first task sorting.
- Import wizard vertical scrolling.
- Metered usage source type persistence.
- Open HA Device navigation context restoration.
- Existing task-specific import entity mapping regression coverage.

## Task Editor Checks

1. Open an existing task in the task editor.
2. Confirm Maintenance Rule #2 uses user-facing wording:
   "A second maintenance rule will be available in a future release. For now, use Maintenance Rule #1 for this task's schedule."
3. Confirm no developer-facing wording about backend rule shapes, phases, or required fields appears.
4. Save an existing task and confirm saved data shape is unchanged.

## Compact Task View Checks

1. Switch Tasks to Compact mode.
2. Confirm compact progress bars show colored fill, not only gray tracks.
3. Confirm progress percentages still match the task progress value.
4. Confirm compact rows remain dense and usable on mobile.

## Urgent Sort Checks

1. Create or identify tasks in each status group: Overdue, Due Today, Due Soon, Healthy/Upcoming, Unknown, Paused, and Completed/Disabled if visible.
2. Select "Most urgent first."
3. Confirm order is: Overdue, Due Today, Due Soon, Healthy/Upcoming, Unknown, Paused, Completed/Disabled.
4. Confirm due date/progress only affects order within the same status priority.
5. Confirm category/name sorting still work when selected.

## Import Wizard Scroll Checks

1. Open a large Task Pack or backup import with enough selected tasks to exceed the viewport.
2. Confirm the wizard header and stepper remain visible.
3. Confirm the middle wizard content scrolls vertically.
4. Confirm the footer with Cancel/Back/Next/Import remains visible and usable.
5. Repeat on mobile width.

## Metered Source Type Checks

1. Create a Metered usage task.
2. Choose "Reset/session counter - add positive deltas."
3. Save and reopen the task editor.
4. Confirm the meter source type still shows "Reset/session counter - add positive deltas."
5. Complete or refresh the task and confirm metered usage sensor values align with session counter behavior.
6. Repeat with "Cumulative meter - already total" and confirm existing cumulative behavior is unchanged.

## Open HA Device Checks

1. From the task list, set filters, sort, and Comfortable or Compact mode.
2. Open a task detail dialog.
3. Press Open HA Device.
4. Press the browser/Home Assistant Back button.
5. Confirm HMM returns to the prior tab, filters, sort, view mode, and task detail context when practical.
6. Confirm normal Home Assistant navigation still works.

## Import Mapping Regression Checks

1. Confirm `python3 -m unittest discover tests` still passes the task-specific entity mapping tests.
2. Manually import two selected tasks that use the same placeholder and map them to different real HA entities.
3. Confirm each imported task keeps its own mapping.
