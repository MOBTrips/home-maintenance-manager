# v0.7.4 Phase 3 Task List Test Plan

Phase 3 refreshes the Tasks tab and adds Comfortable/Compact view mode. It must not change backend behavior, task editor behavior, history behavior, import wizard behavior, storage, imports, exports, NFC handling, or task service APIs.

## Scope

- Comfortable task cards use shared status chip, category icon, progress, and clearer hierarchy.
- Compact view renders dense task rows.
- View mode toggle supports Comfortable and Compact.
- Selected view mode persists across page reloads with browser local storage.
- Existing task list filtering, category grouping, sorting, and actions remain available.

## Comfortable mode checks

1. Open the Tasks tab.
2. Select Comfortable mode.
3. Confirm task cards show:
   - Category icon
   - Task name
   - Status chip
   - Category
   - NFC indicator when a task has NFC tags
   - Progress bar and percentage
   - Next due and last completed dates
4. Confirm each card action still works:
   - Mark complete
   - Snooze 7 days
   - Open HA Device when available
   - View
   - Edit
5. Confirm task editor Delete behavior is still available when editing an existing task.

## Compact mode checks

1. Select Compact mode.
2. Confirm task rows show:
   - Category icon
   - Task name
   - Status chip
   - Compact progress indicator
   - Complete action
3. Confirm compact rows show significantly more tasks per screen than Comfortable mode.
4. Confirm compact row actions still work:
   - Done / complete
   - Snooze
   - Device when available
   - View
   - Edit
5. Confirm rows remain readable on mobile width and action buttons wrap cleanly.

## Preference checks

1. Select Compact mode.
2. Reload the panel.
3. Confirm Compact mode remains selected.
4. Select Comfortable mode.
5. Reload the panel.
6. Confirm Comfortable mode remains selected.

## Filter and grouping checks

1. Filter by category and confirm the selected mode still applies.
2. Filter by status and confirm the selected mode still applies.
3. Sort by Most urgent, Category, and Name.
4. Confirm grouped category headings remain visible when grouping is active.
5. Confirm the empty filtered state still renders when no tasks match.

## Non-task-list regression checks

1. Confirm the Dashboard still renders.
2. Confirm History still renders.
3. Confirm NFC Tags still render.
4. Confirm Settings import/export controls still render.
5. Confirm the import wizard still opens and navigates.

python3 -m compileall custom_components/home_maintenance_manager
