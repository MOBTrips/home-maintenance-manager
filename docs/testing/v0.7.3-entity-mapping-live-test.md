# v0.7.3 Entity Mapping Live Test Checklist

Use this checklist to manually test the Home Maintenance Manager v0.7.3 Task Pack entity mapping workflow in a live Home Assistant browser session.

## Test Environment Setup

- Home Assistant is running with HMM v0.7.3 installed.
- HMM integration is configured under **Settings -> Devices & services**.
- The **Maintenance** sidebar panel opens without frontend errors.
- At least one built-in Task Pack with entity requirements is available, such as HVAC, Hot Tub, Pool, Sump Pump, or Generator.
- A Full Coverage Test Pack is available for upload if you are testing synthetic edge cases. It should include required and optional requirements, runtime and meter rules, linked entities, sparse metadata, rich metadata, and multiple tasks using one requirement.
- A recent Home Assistant backup or HMM JSON export exists before destructive regression checks.

## Browser And Cache Reset

Steps:

1. Open Home Assistant in a fresh browser tab.
2. Hard refresh the page.
3. If the old wizard still appears, clear site data for the HA origin or open a private/incognito window.
4. Confirm the Maintenance panel loads the v0.7.3 wizard text, including "Map entity requirements".

Expected result:

- The browser uses the new frontend JavaScript and does not show the older single-panel mapping UI.

## Known Risk Areas

- HA entity registry metadata may be missing, especially area/device names.
- Sparse requirements may show no suggestions by design.
- Browser cache may load old frontend JavaScript.
- Required unresolved entities should pause affected tasks, not break import.
- Optional unresolved entities should not block import.
- Selection state should not reset between wizard steps.
- Entity picker behavior depends on Home Assistant frontend components being available in the panel context.
