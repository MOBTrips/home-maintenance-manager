# v0.7.4 Phase 6 History Screen Test Plan

Phase 6 refreshes the History screen only. It must not change backend behavior, completion behavior, task data, filters, dashboard, task list, task detail dialog, task editor, imports, exports, task packs, or the import wizard.

## Scope

- History entries render in a timeline-style layout grouped by date.
- Existing completion history and activity history records are displayed without introducing a new schema.
- Category icons and supported status chips are used where existing data supports them.
- Empty history state is clear and intentional.

## Data Checks

1. Complete a task from the dashboard or task list and confirm the completion appears in History.
2. Complete a task from the task detail dialog and confirm the completion appears in History.
3. Confirm existing `completion_history` records appear as completed timeline entries.
4. Confirm existing `activity_history` records appear as activity timeline entries.
5. Confirm duplicate completion/activity records for the same completion do not create confusing duplicate rows.
6. Confirm no task data migration or new history fields are introduced.

## Timeline Checks

1. Confirm entries are grouped by local calendar date.
2. Confirm newest date groups appear first.
3. Confirm newest entries inside each date group appear first.
4. Confirm each entry shows task name, category, activity label, time, and notes when available.
5. Confirm category icons render for supported categories.
6. Confirm completed and snoozed records show an appropriate status chip.

## Existing Behavior Checks

1. Confirm existing History tab navigation still works.
2. Confirm existing task completion behavior is unchanged.
3. Confirm any existing History filtering behavior still works if present in the current panel state.
4. Confirm dashboard, task list, task detail dialog, task editor, Settings import/export, task packs, and import wizard still render.

## Empty State Checks

1. Use an install or fixture with no completion or activity history.
2. Confirm the History screen shows a clear empty state.
3. Confirm no errors appear in the browser console.

## Layout Checks

1. Verify desktop layout is readable and grouped by date.
2. Verify mobile layout stacks cleanly without overlapping text or controls.
3. Verify long task names and notes wrap or truncate cleanly.
4. Verify the timeline remains readable with many entries.

## Validation Checks

Run:

```bash
python3 -m compileall custom_components/home_maintenance_manager
```

Run the frontend parse check with the bundled or local Node runtime:

```bash
node --check custom_components/home_maintenance_manager/frontend/home-maintenance-manager-panel.js
```

Run the existing test suite if `pytest` is available:

```bash
python3 -m pytest
```
